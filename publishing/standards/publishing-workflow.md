# Complete Publishing Workflow

## Research

Define the learner, outcome, scope, platform, parts, safety boundaries, and search intent. Verify technical facts with primary documentation. Record official SunFounder tutorial, documentation, and video URLs; do not copy their instructions.

↓

## Draft

Start from the matching template. Write original copy, add centralized product references, create accessible original visuals, and keep pins, values, diagrams, and code consistent.

↓

## Review

Complete the relevant content checklist, SEO checklist, and publishing checklist. Verify electrical accuracy, official resources, affiliate behavior, internal links, mobile layout, and accessibility. Run configured checks, the production build, and build audit.

↓

## Publish

Move the reviewed content and assets into the established website locations only after every required field and asset is ready. Rebuild and smoke-test the generated local route.

↓

## Git Commit

Review the diff, exclude unrelated changes, and create a focused commit with a descriptive message. Committing requires explicit authorization when the publishing task does not already include it.

↓

## Push

Push the intended branch only with authorization. Never push automatically merely because validation passed.

↓

## Cloudflare Deploy

Confirm the Cloudflare build and deployment succeed. Smoke-test the live canonical page, responsive images, diagrams, official resources, internal links, metadata, and affiliate buttons. Record genuine follow-up work without using fake urgency or presenting unfinished content as complete.
