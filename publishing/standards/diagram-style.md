# Diagram Style

Diagrams teach connections, not decoration. Use an uncluttered left-to-right or top-to-bottom signal path, standard schematic symbols where practical, and labels a beginner can read without zooming.

Circuit diagrams must identify board pins, component values, polarity, power, and ground. Breadboard diagrams must distinguish connected rows, show component placement, and make series or parallel relationships visually clear. Pin numbers must match the code and prose exactly.

Create original editable SVGs with a useful `viewBox`, descriptive `title` and `desc`, sufficient contrast, and responsive dimensions. Do not reproduce vendor diagrams.
