# SEO checklist

## Search intent and page focus

- [ ] One primary query and a small set of closely related terms are documented.
- [ ] Content type matches intent: project, explanatory tutorial, or buying guide.
- [ ] The page offers original practical value rather than repeating generic summaries.
- [ ] Key terms appear naturally; repetition is not forced.

## Metadata and structure

- [ ] Title is unique, specific, and consistent with the H1.
- [ ] Meta description is unique, accurate, and approximately 120–160 characters.
- [ ] Page contains exactly one H1 with logically nested H2/H3 headings.
- [ ] Canonical URL uses the production origin and preferred trailing-slash format.
- [ ] Open Graph and Twitter metadata use an appropriate image, alt text, and dimensions.
- [ ] `publishedDate` is preserved and `updatedDate` reflects a meaningful revision.

## Discovery and internal links

- [ ] Breadcrumbs reflect the visible site hierarchy and match Breadcrumb JSON-LD.
- [ ] Project pages link to previous/next builds, related tutorials, their kit, a buying guide, and related projects.
- [ ] Anchor text describes the destination; important links are standard crawlable anchors.
- [ ] No broken, redirected, placeholder, or orphaned internal link remains.
- [ ] The canonical URL appears in the XML sitemap with the correct `lastmod` when applicable.

## Media and structured data

- [ ] Every meaningful image has descriptive alt text and accurate width/height.
- [ ] Below-the-fold images load lazily; hero media is appropriately prioritized.
- [ ] WebP or AVIF is used for photographs when practical.
- [ ] Article, Breadcrumb, and Product structured data describe visible, accurate page content.
- [ ] No unverified offer, rating, review, price, or availability is added to Product data.
- [ ] Representative structured data passes the Rich Results Test before launch.

## Measurement

- [ ] Primary query, page type, publication date, and URL are entered in the analytics tracker.
- [ ] Search Console indexing and sitemap status are checked after deployment.
- [ ] A 30-, 60-, and 90-day performance review is scheduled.
