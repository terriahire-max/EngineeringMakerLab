# Diagram checklist

## Electrical accuracy

- [ ] Diagram was created from the final reproduced circuit, not an early prototype.
- [ ] Board model, pin names, voltage rails, grounds, polarity, and component values match the article and code.
- [ ] Level shifting, current limiting, flyback protection, common ground, and external power are shown where required.
- [ ] Wire crossings and connected junctions are unambiguous.
- [ ] A second reviewer traced every connection from source to destination.

## Communication quality

- [ ] Layout follows signal or power flow and avoids unnecessary wire crossings.
- [ ] Labels remain readable at the expected mobile width.
- [ ] Color reinforces meaning but is never the only indicator.
- [ ] Power, ground, signal, polarity, and pin numbers use consistent notation.
- [ ] The caption states the diagram's purpose and any important limitation.
- [ ] The article provides a written connection description alongside the diagram.

## Production

- [ ] Editable source is stored separately from the web export.
- [ ] SVG is used when practical; raster exports include explicit dimensions.
- [ ] Fonts are embedded or converted safely, and no editor artifacts remain.
- [ ] Filename uses lowercase kebab-case and the approved asset folder.
- [ ] Alt text identifies the board, component, critical pins, and signal relationship.
- [ ] Export is reviewed in light and dark browser contexts for contrast and legibility.
